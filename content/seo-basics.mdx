# Googleに「発見」してもらうために

SEO（検索エンジン最適化）は、マーケターだけの仕事ではありません。
サイトの構造を作るエンジニアにしかできないSEO対策があります。

## 1. セマンティックHTMLを使う

`<div>` ばかり使っていませんか？
適切なタグを使うことで、Googleはコンテンツの意味を理解しやすくなります。

*   ヘッダーには `<header>`
*   記事本文には `<article>`
*   メインコンテンツには `<main>`

## 2. メタデータを設定する

`<head>` タグ内の情報は超重要です。
Next.jsなら `metadata` オブジェクトで簡単に設定できます。
`title` と `description` は、検索結果にそのまま表示される「お店の看板」です。

## 3. Core Web Vitals (パフォーマンス)

Googleは「表示が遅いサイト」の順位を下げます。
特に **LCP** (Largest Contentful Paint) は重要です。
画像最適化（Next.jsの `<Image />`）や、不要なJavaScriptの削減などで高速化しましょう。

## 結論

技術的なSEO対策は「サイトの土台」です。
土台がしっかりしていないと、どんなに良い記事を書いても埋もれてしまいます。
