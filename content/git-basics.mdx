まさか `file_final_v2_fix.zip` なんてファイル名で管理していませんか？
エンジニアなら **Git** を使いましょう。

## Gitの基本コマンド3神器

### 1. git add
「このファイルを保存対象にするよ」という宣言。
ステージングエリアに乗せるとも言います。

### 2. git commit
「セーブポイントを作成する」作業。
必ずメッセージをつけて、何をしたかを記録します。
`git commit -m "バグ修正"`

### 3. git push
「自分のPCからGitHub（インターネット上）へアップロード」する作業。
これでPCが爆発してもデータは無事です。

## GitHubを使うメリット

*   **バックアップ**: 世界最強のサーバーにコードを保存。
*   **ポートフォリオ**: 「これだけコードを書きました」という証明になる。
*   **コラボレーション**: 世界中のエンジニアとコードを共有できる。

まずは個人の開発で慣れていきましょう。
